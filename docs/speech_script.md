# 四六级单词训练与 C++ 桌宠伴学系统发言稿

## 第 1 页：项目介绍

各位老师好，我的大作业题目是《四六级单词训练与 C++ 桌宠伴学系统》。

这个项目不是一个单独网页，而是一个 Windows C++ 桌面应用。程序入口是 `DesktopPet.exe`，它会同时启动一个 C++ 透明悬浮桌宠窗口，并用 WebView2 内嵌一个背单词学习界面。网页负责展示和交互，真正的朗读、提醒、记录、桌宠反馈等能力由 C++ 完成。

## 第 2 页：问题与目标

我最开始想解决三个学习场景里的问题：第一，单词朗读不能让网页卡住；第二，查过的词需要进入后续练习计划；第三，错题和到期复习需要更明显的提醒。

所以项目目标分成两层：前端界面负责练习、查词、统计和计划展示；C++ 负责桌面桌宠、TTS 朗读、本地记录、多线程提醒和消息处理。这样既有可用的学习界面，也能体现 C++ 大作业的技术点。

## 第 3 页：总体架构

整个项目的主体是 `DesktopPet.exe`。程序内部有三个核心部分：左边是 WebView2 内嵌的学习界面，中间是 C++ 主程序，右边是 C++ 桌宠能力模块。

网页点击朗读、答题、加入复习等操作后，会通过 WebView2 的 `postMessage` bridge 发送给 C++。C++ 收到后调用对应能力，比如 SAPI 朗读、保存记录、弹出桌宠气泡。这样网页不是外部浏览器页面，而是嵌在 C++ 程序里的界面。

## 第 4 页：C++ 核心实现

C++ 部分主要用了这些知识点。

第一是 Win32 窗口和消息循环，用来创建桌宠窗口、右键菜单和热键。第二是 Direct2D 和 WIC，用来绘制桌宠图片、阴影和气泡。第三是 Windows SAPI TTS，用来朗读英文单词，并且放到后台线程里执行，避免阻塞界面。

另外项目还用了 Boost.JSON 解析配置和消息，多线程处理 HTTP 服务、朗读和提醒，文件读写保存学习记录、日志和窗口位置。

## 第 5 页：WebView2 内嵌与桥接

这一页是项目整合的关键。

原来如果网页和 C++ 分开，就只能通过外部浏览器请求本地服务，看起来不够像一个整体。现在我改成 C++ 程序创建 WebView2 窗口，直接加载项目里的 `web/index.html`。

网页里调用 C++ 时，不再依赖外部浏览器协议，而是通过 `chrome.webview.postMessage` 把请求发给 C++。C++ 根据路径执行 `/status`、`/speak`、`/record`、`/study-state` 等逻辑，再把 JSON 结果返回给网页。

## 第 6 页：学习计划与数据流

学习流程从查词或答题开始。网页会更新当前账号的学习进度，然后根据正确率、错题、查词计划和复习时间计算下一步练什么。

规划优先级是：到期复习优先，其次是查词计划，再是错题专项，最后是今日新题。答题结果也会同步给 C++，C++ 可以保存记录并通过桌宠气泡提醒用户。

## 第 7 页：运行与验证

运行时只需要双击或执行 `open_web.bat`。这个脚本会构建并启动 C++ 主程序。程序启动后会出现 C++ 桌宠窗口和内嵌学习窗口。

我做过三类验证：第一，`node --check web/app.js` 检查网页脚本语法；第二，CMake 构建 C++ 项目通过；第三，运行程序后 `/status` 能返回 `ok`，说明 C++ 服务和桌宠能力已经启动。

项目也已经上传到 GitHub，GitHub Pages 可以作为前端展示入口；完整演示则运行本地 C++ 程序。

## 第 8 页：总结

总结一下，这个项目的重点不是做一个单独网页，而是用 C++ 桌面应用承载学习体验。

目前已经完成 C++ 桌宠主程序、WebView2 内嵌学习界面、朗读、提醒、记录、桥接，以及去除重复 HTML 桌宠。后续如果继续扩展，可以加入更完整的词库导入、更细的间隔重复算法、更多桌宠皮肤动作，以及学习数据导出报告。
